Le principe des 3 zones
Fichier créé/modifié  →  git add  →  git commit -m "message"
   (Untracked)          (Staged/prêt)      (Sauvegardé dans l'historique)
git status — observer l'état (ne modifie rien)
Dépôt vide, aucun fichier :
nothing to commit
→ Rien à signaler, dossier propre, aucun fichier créé.
Fichier créé, mais pas encore add :
Untracked files:
        test.txt
→ "Untracked" = non suivi. Git voit le fichier mais ne le surveille pas encore.
Après git add :
Changes to be committed:
        new file:   test.txt
→ Le fichier est "staged" (préparé), prêt à être sauvegardé au prochain commit.
Après git commit :
nothing to commit, working tree clean
→ "working tree clean" = tout est sauvegardé, rien n'est en attente.
git add nom_fichier — préparer un fichier pour la sauvegarde
Fait passer un fichier de "Untracked" à "Changes to be committed".
git commit -m "message" — sauvegarder définitivement
Règles obligatoires :

Le -m est obligatoire pour indiquer que ce qui suit est un message (sans lui, Git pense que c'est un nom de fichier)
Le message ne peut jamais être vide

Exemple de retour après un commit réussi :
[master (root-commit) 2fcbe66] premier commit
 1 file changed, 1 insertion(+)
 create mode 100644 test.txt

master → la branche
(root-commit) → premier commit du dépôt
2fcbe66 → identifiant unique du commit (hash)
1 file changed, 1 insertion(+) → résumé du changement
create mode 100644 test.txt → confirme la création du fichier
